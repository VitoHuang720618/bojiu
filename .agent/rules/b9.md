- 環境與端口 (Environment):
  - Manager Backend: http://localhost:3002
  - Manager Frontend: http://localhost:3001
  - Demo Site: http://localhost:3000 

- 開發規範 (Coding Standards):
  - 框架: 使用 **Vue 3 + Script Setup + TypeScript**。
  - 樣式: 使用 Vanilla CSS (Scoped)，不依賴 Tailwind。
  - RWD 適配規範:
    - **Mobile (< 740px)**: 
      - 單欄佈局，重點優化觸控體驗。
    - **Tablet (740px ~ 1279px)**: 
      - 這是最複雜的區間，必須使用流體佈局 (Fluid Layout，使用 % 而非固定 px)。
      - 需重點測試對象：
        - **iPad Mini (768px)**: 接近手機寬度的平板。
        - **iPad Air (820px)**: 常見的中型平板。
        - **iPad Pro (1024px)**: 接近小筆電的大型平板。
    - **Desktop (>= 1280px)**: 
      - 寬螢幕佈局，最大內容寬度限制為 1200px 或 1500px。
  - 資源處理: 
    - 圖片路徑必須統一使用 helper function 處理。
    - 參考實作: 位於 `demo/src/utils/imageHelper.ts` (或專案中定義該函數的相對位置) 的 [getImageUrl](cci:1://file:///Users/vitohuang/project/b9-website-recreation/manager/front/src/components/ConfigManager.vue:1291:0-1309:1)。
    - 目的: 確保能動態切換本地開發資源與 API 遠端資源。

- 工作流程 (Workflow):
  - **Commit 策略**: 完成一個具體的修改任務後（如修復 Bug、完成新功能），確保程式碼無報錯，請主動執行 git commit。
  - Commit Message: 使用中文，格式清晰（例：`Fix: 修正平板首頁跑版問題`）。
